## ✅ SoilDbExecutor FetchSize動的設定対応 テストケース一覧

| 大項目       | 中項目   | 小項目             | テストパターン（アクション）                     | 期待値                                           |
| ------------ | -------- | ------------------ | ------------------------------------------------ | ------------------------------------------------ |
| 設定値取得   | 正常系   | 通常値設定         | CPM_CMM_CRYPTIC_CONTROLにVALUE=300を登録して実行 | JdbcTemplateにfetchSize=300が設定される          |
| 設定値取得   | 異常系   | 不正値（文字列）   | VALUE='abc'を登録して実行                        | fetchSizeはデフォルト10が設定される              |
| 設定値取得   | 異常系   | 不正値（0）        | VALUE='0'を登録して実行                          | fetchSizeは10固定となる                          |
| 設定値取得   | 異常系   | 不正値（負数）     | VALUE='-5'を登録して実行                         | fetchSizeは10固定となる                          |
| 設定値取得   | 異常系   | レコード未登録     | SOIL_DB_FETCH_SIZEが存在しない状態で実行         | fetchSizeは10固定となる                          |
| キャッシュ   | 正常系   | 初回読込キャッシュ | getFetchSize()を2回連続呼出                      | 2回目はDBアクセスせずキャッシュ値を返す          |
| キャッシュ   | 正常系   | キャッシュクリア   | clearCache()呼出後にgetFetchSize()実行           | DBから再取得し、新しい値を反映する               |
| 共通動作     | 正常系   | 既存メソッド互換   | 旧メソッド query(sql, param, rch) を呼出         | 従来通り動作し、fetchSizeが設定される            |
| 共通動作     | 正常系   | 個別上書き指定     | query(sql, param, rch, 200) を呼出               | fetchSize=200で実行される                        |
| 共通動作     | 異常系   | 個別上書きに不正値 | query(sql, param, rch, 0) を呼出                 | fetchSizeはSoilDbConfigの値（例:10）が使用される |
| 性能・安定性 | 性能確認 | 大量データ取得     | 10万件超のSELECTをfetchSize=300で実行            | 正常に処理完了し、CPU高騰が発生しない            |
| 性能・安定性 | 回帰確認 | CLOB列含むSELECT   | CLOBカラムを含むSQLをfetchSize=10で実行          | CPU異常上昇が発生せず、従来動作を維持            |
